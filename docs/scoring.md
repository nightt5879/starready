# Scoring Model

StarReady scores the public repository surface from 0 to 100. The score is intentionally practical: it rewards signals that help a visitor understand, trust, and try a project quickly.

## Categories

| Category | Weight |
| --- | ---: |
| First impression | 28 |
| Product clarity | 20 |
| Trust | 24 |
| Engineering | 18 |
| Community | 10 |

## Principles

- A clear README beats clever copy.
- A runnable quickstart beats a long explanation.
- Trust signals should be visible before a user installs anything.
- Community files matter because they reduce contributor hesitation.
- The score is a launch-readiness proxy, not a promise of GitHub stars.

## Recommended Gates

| Gate | Suggested threshold |
| --- | ---: |
| Personal project before public launch | 70 |
| Serious open-source release | 80 |
| Organization template or flagship project | 85 |
